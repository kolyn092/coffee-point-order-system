- 기대 결과: Consumer 수 1·2·3을 각각 세 번 실행하고, 처리량, partition별·합계 lag,

## 재현 절차

1. 전용 Docker Compose 환경을 기동한다.
2. `bash ./scripts/load-test/invoke-load-test.sh --scenario consumer-scaling`을 Consumer 수 1·2·3에 각각 세 번 실행한다.
3. 각 실행의 `summary.json`, `report.md`, 중앙 비교 보고서를 확인한다.

- PR #42 본문은 전체 Gradle 테스트가 60초 제한을 넘어 완료 결과를 확인하지 못했고,
  `consumer-scaling`은 미실행이라고 명시한다.
- 2026-07-19 21:20 KST P1 리뷰는 결과 보고서 또는 PR에서 확인 가능한 요약·첨부가 없다고 지적했다.
- 현재 브랜치의 `scripts/load-test/invoke-load-test.sh`에는 사용자 작업 중인 변경이 있어 이 기록은 수정하지 않았다.

- 가설: 자동화 스크립트와 설정이 있으면 Consumer 확장 성능 검증이 완료됐다.

  - 결론: 기각.

## 근본 원인

필요한 실행 환경과 시간이 확보되지 않아, 구현 검증을 정적 검사와 일부 단위 테스트까지만 수행했다.

각 Consumer 수의 중앙값·최댓값 비교와 판정을 PR에 연결해야 한다.
