## 가설과 검증

- 가설: 순위와 주문 수만 맞으면 인기 메뉴 응답 계약이 보장된다.
  - 검증: `name`, `price`가 누락되거나 잘못 매핑돼도 기존 테스트가 통과한다.
  - 결론: 기각.

## 근본 원인

API 응답 DTO의 일부 필드만 assertion해 응답 변환의 전체 계약을 테스트로 보호하지 못했다.

## 해결 또는 완화

대표 Top 3 항목의 메뉴 ID, 이름, 가격, 주문 수를 함께 검증하도록 통합 테스트를 보강했다.

## Before/After 검증

- Before: 메뉴명·가격 매핑 오류를 테스트가 검출하지 못했다.
- After: 응답의 모든 계약 필드를 확인하고, 재리뷰에서 P0 M4의 정렬·만료·장애 응답까지 함께 확인했다.

- Top 3 각 응답의 `menuId`, `name`, `price`, `orderCount`
- 동점 정렬과 빈 배열
